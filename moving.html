moving.js

Moving things around in a website is not a very common task. But for a web application, it is very handy and open a lot of possibilities. The DOM had already everything needed in order to provide that interaction. No need to load a framework for that. Just provide a function to store the object's position in case persistance is required.

In order to have a movable object, there is only need to include a script tag at the end of the <body> and add a class for the object that is going to move. All css is preserved and there is a small specific markup required: the objects have to be absolutely positioned.

Use case:

<body>
	<div id="f1" class="small-box" style="top: 325px; left: 120px;">I'm fixed</div>
	<div id="m1" class="small-box movable" style="top: 325px; left: 220px;">Move me</div>
	<div id="m2" class="small-box movable" style="top: 325px; left: 320px; background-color:#cce;">Move me too!</div>

			(...)

	<script type="text/javascript" src="javascripts/moving.js"></script>
	<script>
		var store_changes = function(changes) {
			console.log("storing",changes);
			return 0;
		}
		moving.init(store_changes);
	</script>
</body>
