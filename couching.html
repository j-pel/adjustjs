couching.js

This is an easy to use and fully asynchronous frontend to manage a CouchDB instance using ES6 promises.

How to use

The following is a step by step guide to test the functionality included in couching.js. Just press the "Next" button to start.

CouchDB connection:Admin user:Password:
 

Before starting

Couching requires access to a CouchDB instance. Installing and running CouchDB is straightforward. Once installed, it will listen to port 5984 by default. It is possible also to run CouchDB as a service from a cloud provider. Some of them offers CouchDB for free for low traffic applications.

Couching initializer returns an object instance with the database name and uri. If the database does not exist, it must be created before using it. Only an user with admin role can create one, so a succesful login is needed. Press the "Next" button in order to log into the database.
